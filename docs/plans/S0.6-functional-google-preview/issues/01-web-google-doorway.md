## Comments

**2026-07-17 ‚Äî the layering test made a design decision, which is what it is for.** `layering.test.ts` bans `@react-native-google-signin/` imports outside a 6-file allowlist whose comment says adding to it "is a real decision‚Ä¶ it cannot happen by reflex." Ticket 02 puts Google's *component* in a file that draws pixels ‚Äî exactly what the rule forbids. Considered: re-exporting a React component through the repository (drags UI through a data-access module to keep the list at six), or skipping the official button (reverses a founder ruling). Took the honest option: `GoogleSignInButton.native.tsx` joins the seam, with the reasoning recorded in the allowlist. The web half needs no entry ‚Äî it renders an iframe and imports nothing. The rule worked: it forced the decision into the open instead of letting an `import` slide through review.

**2026-07-17 ‚Äî the shared prop contract, and why both platforms take props they ignore.** The two doorways have different *shapes*: native pulls (`onPress` ‚Üí await ‚Üí done), web pushes (GIS owns the click; the credential lands in a callback). Declaring props per-file would mean the screen is typechecked against whichever Metro resolved first, with the other platform's mismatch surfacing as a runtime nothing on the platform nobody was looking at. `googleSignInButtonContract.ts` mirrors `authContract.ts`'s precedent ‚Äî types once, implementations fork. Review flagged this as close to Refused Bequest (each side uses a disjoint subset) and rated it defensible-but-marginal; the alternative, a union the screen narrows, puts `Platform.OS` back in UI code that S0.5 worked to remove.

**2026-07-17 ‚Äî TDD caught a test that would have been flaky in CI and deterministic here.** The first version of "stores the session exactly as password sign-in does" asserted two `Date.now()`-derived `expiresAt` values were equal. It failed by **2ms**. The code was right; the assertion was asserting the clock stood still. Fixed to compare everything the response determines and bound the expiry to a range. Worth recording because the failure mode is the dangerous kind ‚Äî it would have passed on a faster machine and been "fixed" later by re-running CI until green.

**2026-07-17 ‚Äî `localStorage` is stubbed rather than switching Jest presets.** These suites run under jest-expo's native preset, whose `window` has no `localStorage`. The "proper" fix is a web-preset project ‚Äî the multi-project Jest config S0.5 weighed and deferred as "a test-infrastructure decision far larger than the feature it would serve." That call still holds and belongs to the web-surface story. Stubbing the module's one browser dependency buys the same assertions for a few lines. **Honest limit:** it proves the module's logic, never that a browser agrees ‚Äî the container run (ticket 03) is what proves the browser.

**2026-07-17 ‚Äî code review: the `disabled` prop was decoration, not a guard (both axes, independently, ranked it first).** `pointerEvents: 'none'` on the host `View` cannot stop a click reaching GIS's **cross-origin iframe**, which handles its own events. So while an email sign-in was in flight, a founder could plausibly still trigger Google: `onStart` would stomp `busy='email'` and two sign-ins would race for one session. The fix moves the guard to where the decision is actually made ‚Äî at the credential ‚Äî and reads the prop through a ref, because GIS registers one callback for the page's life and a captured boolean would be the value at mount forever. That distinction (a callback, not a boolean) is the difference between a guard and a decoration; both are now pinned by tests.

**2026-07-17 ‚Äî the review's other finding forced a better shape: `googleCredentialHandler.ts`.** The web button held the diff's trickiest logic (ref-latched handlers, the cancelled flag, the double `.catch`) and was the only new file with no test ‚Äî precisely the S0.2 `getTokens()` shape this repo keeps citing. A component test would have meant `@testing-library/react-native`, a new dev dependency *and* a reversal of a documented scope decision (`jest.config.js`: "no component-snapshot theatre", 06b ¬ß7) inside a story whose boundary says nothing about test infrastructure. The resolution: the logic never had to be in a component. Extracting it tests the real thing with no new dependency, leaves the component as wiring, and ‚Äî a second win review had flagged separately ‚Äî takes the `.web`-suffixed import out of UI code, moving it to an auth-seam module where naming a platform file is already the convention.
